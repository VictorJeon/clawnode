# OpenClaw 설치 대행 고객 대응 SOP (내부 운영용)

> 대상: Mason (서비스 운영자)
> 목적: 주문 접수부터 사후관리/리뷰 요청까지 누락 없이 처리

---

## 0) 기본 원칙 (먼저 확인)

- 고객이 어렵게 느끼지 않게 **짧고 명확하게 안내**
- 민감정보(토큰/API 키/서버 비밀번호)는 **DM으로만 수신**
- 단계별로 **체크리스트 완료 후 다음 단계 이동**
- 이슈 발생 시 10분 이상 지연되면 바로 중간 안내 메시지 전송

---

## 1) 주문 접수 (Kmong)

**예상 소요:** 10~15분

### 체크리스트
- [ ] 주문자명 / 연락 채널 확인 (Kmong, 카톡, 텔레그램)
- [ ] 티어 확인 (STANDARD / DELUXE / PREMIUM)
- [ ] 희망 일정 1~2개 수집
- [ ] OS 확인 (Mac / Windows)
- [ ] DELUXE/PREMIUM일 경우 서버 진행 여부 확인

### 초기 안내 템플릿 (Kmong)
```
안녕하세요, 주문 감사합니다 🙂
주문하신 상품은 [STANDARD/DELUXE/PREMIUM]으로 확인되었습니다.

진행 전 준비 가이드를 먼저 보내드릴게요.
가이드 체크 후 "준비 완료"라고 답주시면,
가장 빠른 시간으로 설치 일정을 확정해드리겠습니다.
```

---

## 2) 사전 준비 안내

**예상 소요:** 고객 준비 10~30분 (운영자 안내 5분)

### 체크리스트
- [ ] `customer-prep-guide.md` 전달
- [ ] 고객에게 본인 티어에 해당하는 준비 항목만 다시 요약 전달
- [ ] 민감정보 전달 방식 안내 (DM, 전달 후 삭제 권장)
- [ ] 설치 일정 잠정 홀드

### 전달 템플릿 (카톡/Kmong)
```
설치 전에 아래 준비만 완료해주시면 바로 진행 가능합니다.
(가이드 첨부)

핵심 3가지만 먼저 체크해주세요:
1) 텔레그램 봇 생성 + /start 전송
2) 사용하실 AI 모델 준비
3) [DELUXE/PREMIUM만] 서버 정보 준비

준비 끝나면 "준비 완료"라고 보내주세요.
```

### 준비 지연 대응 (고객이 준비를 안 해옴)
- 24시간 무응답: 1차 리마인드
- 48시간 무응답: 2차 리마인드 + 일정 재조율
- 72시간 무응답: 주문 상태 확인 요청 (계속/보류 선택)

리마인드 템플릿:
```
안녕하세요 🙂 설치 진행을 위해 준비 상태 확인차 연락드립니다.
가능하실 때 "준비 완료" 또는 "일정 재조율"로 답주시면 바로 맞춰드릴게요.
```

## 3) 설치 진행 (티어별 분기)

## 3-1. STANDARD (작업 1일)

**예상 소요:** 60~90분

### 체크리스트
- [ ] 구글밋 접속
- [ ] 고객 PC에서 설치 파일 실행 (.command / .bat)
- [ ] 의존성 설치 완료 확인
- [ ] 텔레그램 연동 테스트 (/start, 질의 응답)
- [ ] 기본 보안 점검 (토큰/키 노출 여부, 접근권한)
- [ ] 설치 결과 수집 (버전/경로/실행 상태)
- [ ] 에이전트 핸드오프 정보 정리

### 진행 멘트 템플릿
```
지금부터 설치를 시작하겠습니다.
중간에 비밀번호/권한 허용 창이 뜨면 제가 말드리는 시점에만 눌러주세요.
전체 1시간 내외로 끝납니다.
```

---

## 3-2. DELUXE (작업 3일)

**예상 소요:** 총 1.5~3시간 (분할 진행 가능)

STANDARD 전체 + 아래 추가

### 체크리스트
- [ ] VPS 세팅 진행 (`vps-setup-sop.md` 기준)
- [ ] 도메인 연결
- [ ] SSL 적용 확인 (https)
- [ ] 방화벽 기본 정책 적용
- [ ] 24시간 상시 운영 상태 확인
- [ ] 장애 대응 연락 채널 확정

### 추가 안내 멘트
```
DELUXE는 로컬 설치 + 서버 상시 운영까지 포함됩니다.
오늘은 서버 안정화까지 진행하고,
최종 점검 결과를 체크리스트로 전달드리겠습니다.
```

---

## 3-3. PREMIUM (작업 5일)

**예상 소요:** 3~5일 (일정 합의 필수)

DELUXE 전체 + 아래 추가

### 체크리스트
- [ ] 3시간 1:1 바이브코딩 강의 일정 확정
- [ ] MVP 요구사항 수집 (필수 기능/우선순위)
- [ ] 구현 범위 확정 문서 공유
- [ ] 앱 제작/실습 (Cursor / Claude Code / Codex)
- [ ] 배포 완료 (Vercel/Netlify)
- [ ] 고객 인수 테스트

### 요구사항 수집 템플릿
```
MVP 제작을 위해 아래 4가지만 먼저 알려주세요.
1) 만들고 싶은 앱 한 줄 설명
2) 꼭 필요한 기능 3개
3) 참고 서비스/디자인 링크 (있으면)
4) 배포 후 실제 사용 대상 (본인/팀/고객)
```

### 필수 전달 항목
- [ ] 고객명 / 티어
- [ ] OS / 설치 방식 (로컬 or VPS)
- [ ] 설치 경로
- [ ] 실행 상태 (정상/경고)
- [ ] 텔레그램 연동 결과
- [ ] AI 모델/키 설정 상태
- [ ] 미해결 이슈
- [ ] 다음 액션 담당자

- 요청사항: {Sol/Nova에게 필요한 후속 작업}
```

## 5) 완료 처리

**예상 소요:** 15~30분

### 체크리스트
- [ ] 기능 테스트 3종 완료
  - [ ] 텔레그램 질의응답
  - [ ] 파일/요약 기능 1회
  - [ ] 재시작 후 정상 동작
- [ ] 고객용 완료 안내문 전달
- [ ] 사용 가이드(`customer-prep-guide.md` 최신본) 재전달
- [ ] Kmong 완료 처리

### 완료 안내 템플릿
```
설치가 완료되었습니다 ✅

오늘부터 바로 사용 가능하며,
초기 사용은 아래 3가지부터 시작해보시면 좋습니다:
1) 오늘 일정 정리
2) 링크/문서 요약
3) 내일 알림 등록

사용 중 불편하신 점은 언제든 메시지 주세요.
(사후관리 기간 내 우선 지원)
```

## 6) 사후 관리 운영 기준

### 기간별 범위
- **STANDARD:** 1주
- **DELUXE:** 2주
- **PREMIUM:** 1개월

### 지원 범위
- 동작 오류 점검/복구
- 기본 사용법 안내
- 간단 프롬프트/자동화 개선
- (PREMIUM) MVP 관련 경미한 개선/피드백

### 제외 범위 (별도 견적)
- 신규 대형 기능 개발
- 외부 서비스 유료 플랜 비용
- 고객 인프라 자체 장애(결제 미납/도메인 만료 등)

### 응답 시간 가이드
- 운영 시간 내 2~6시간 이내 1차 응답
- 긴급 장애는 우선 확인 후 임시복구 안내

### 에스컬레이션 기준
- 30분 내 복구 불가
- 인증/결제/외부 서비스 정책 이슈
- 데이터 유실 가능성

에스컬레이션 템플릿:
```
현재 건은 즉시 복구가 어려워 상위 기술 검토로 전환했습니다.
원인 파악 후 {예상 시간} 내 중간 결과를 먼저 공유드리겠습니다.
```

**권장 타이밍:** 완료 후 24~72시간 (고객이 실제 사용해본 뒤)

### 체크리스트
- [ ] 최소 1회 이상 고객 사용 확인
- [ ] 해결된 포인트 1~2개 리마인드

### 리뷰 요청 템플릿

도움이 되셨다면 Kmong에 짧은 후기 남겨주시면 큰 힘이 됩니다.
특히 어떤 점이 좋았는지 한 줄만 적어주셔도 충분합니다. 감사합니다!
```

## 8) 트러블 상황 대응

## A. 고객 준비 미완료
- 증상: 토큰/키/서버정보 누락
- 처리:
  1) 누락 항목 3줄 요약 전달
  2) 준비 가이드 해당 섹션 다시 공유
  3) 일정 재확정

템플릿:
```
현재 설치에 필요한 정보 중 {누락항목}이 아직 필요합니다.
해당 정보만 전달주시면 바로 이어서 진행하겠습니다.
```

## B. 설치 실패 (환경 문제)
- 증상: 권한 오류, 의존성 충돌, 네트워크 차단
- 처리:
  1) 오류 캡처 수집
  2) 재시도 1회
  3) 우회 경로(원격지원/환경 재설정) 제시
  4) 30분 이상 지연 시 중간 안내

템플릿:
```
현재 설치 중 환경 충돌이 확인되어 우회 방식으로 복구 중입니다.
데이터 손실 없이 진행 중이며, {예상시간} 내 결과 공유드리겠습니다.
```

## C. 환불 요청
- 원칙: 감정 대응 금지, 사실/범위/진행률 기준으로 처리
- 처리:
  1) 요청 사유 확인
  2) 진행된 작업 범위 정리
  3) Kmong 정책 기준 안내
  4) 부분 환불/대체 지원안 제시

템플릿:
```
불편을 드려 죄송합니다.
요청 주신 내용 확인했고, 현재 진행 범위는 아래와 같습니다:
- 진행 완료: {...}
- 미진행: {...}

Kmong 정책 기준에 맞춰 가능한 처리안(환불/대체지원)을 바로 안내드리겠습니다.
```

- [ ] 티어별 약속 범위 전부 이행
- [ ] 사후관리 종료일 캘린더 등록
- [ ] 리뷰 요청 일정 등록
