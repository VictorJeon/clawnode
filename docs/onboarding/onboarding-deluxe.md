## [38:00~45:00] SSH 기초 (7분)

### 핵심 비유

- SSH = "서버 원격 리모컨"
- 터미널 = "텍스트로 조종하는 리모컨 앱"

### Mason 진행 대본

> "SSH는 서버에 원격으로 접속하는 방법이에요."
>
> "비유하면, 집 TV 앞에 없어도 리모컨 앱으로 채널 바꾸는 느낌이에요."
>
> "우리는 터미널이라는 검은 창으로 서버에 들어가요. 무서워 보이지만 할 건 딱 4개 명령어예요."

### 실습 명령어 (화면 공유 + 따라 하기)

```bash
ssh openclaw@<IP>
openclaw gateway status
openclaw gateway restart
openclaw logs --tail 20
exit
```

### 명령어 설명 대본

> "`ssh openclaw@<IP>`는 서버 현관문으로 들어가는 열쇠예요."
>
> "`openclaw gateway status`는 봇 엔진이 켜져 있는지 확인하는 계기판이에요."
>
> "`openclaw gateway restart`는 앱이 버벅일 때 재실행하는 새로고침 버튼이에요."
>
> "`openclaw logs --tail 20`은 최근 활동기록 20줄을 보는 블랙박스 조회예요."
>
> "`exit`는 서버 방에서 나오는 문이라고 생각하시면 돼요."

✋ [고객 실습] 고객이 직접 SSH 접속 → `openclaw gateway status` 실행 → `exit`로 종료

### 실수 방지 멘트

> "오타가 나도 괜찮아요. 다시 입력하면 돼요. 컴퓨터를 망가뜨리는 명령은 지금 안 쓰고 있어요."

### 섹션 전환 멘트

## 고객 예상 질문 & 답변 (최소 8개)

### Q1. 서버를 꺼두면 비용이 아예 안 나가나요?
> "일부 항목은 줄어들 수 있지만, 디스크나 IP 같은 기본 자원 비용은 남을 수 있어요. 그리고 서버를 끄면 봇도 멈춰요. 보통은 켜두고 운영해요."

### Q2. SSH는 해킹당하기 쉬운 거 아닌가요?
> "열쇠(인증)와 잠금(보안설정)을 제대로 쓰면 안전해요. 설치 때 기본 보안 세팅을 잡아뒀고, 비밀번호/키 관리만 잘하시면 돼요."

### Q3. `restart` 하면 데이터가 날아가나요?
> "아니에요. 재시작은 전원을 껐다 켜는 느낌이라서 보통 데이터는 유지돼요. 그래도 중요한 변경 전엔 스냅샷을 추천해요."

### Q4. 로그는 어디까지 봐야 하나요?
> "처음엔 최근 20줄만 봐도 충분해요. 블랙박스 최근 구간만 확인하는 느낌으로 접근하시면 돼요."

### Q5. 스킬을 많이 깔면 느려지나요?
> "필요 없는 스킬을 과하게 쓰면 복잡도는 올라가요. 스마트폰 앱처럼 필요한 것만 설치하는 게 좋아요."

### Q6. SOUL.md를 잘못 바꾸면 어떡하죠?
> "문구를 원래대로 돌리면 바로 복구돼요. 더 안전하게 하려면 수정 전에 스냅샷 찍어두면 돼요."

### Q7. 봇이 갑자기 답이 이상해졌어요. 제일 먼저 뭘 하나요?
> "Telegram에서 먼저 재질문하고, 그대로면 SSH로 `openclaw gateway restart` 하세요. 대부분 여기서 정상화돼요."

### Q8. 업데이트는 얼마나 자주 해야 하나요?
> "월 1회 점검을 권장해요. 큰 이슈 공지가 오면 그때 즉시 업데이트하면 돼요."

### Q9. 제가 개발자가 아닌데 직접 운영 가능할까요?
> "네, 가능해요. 오늘처럼 상태 확인, 재시작, 스냅샷 3가지만 익히면 실운영은 충분히 가능합니다."

### Q10. 비용이 갑자기 오르면 어떻게 확인하나요?
> "Hetzner Billing에서 사용량 추이를 먼저 보고, 새로 켠 서버/리소스가 있는지 확인하면 원인을 빠르게 찾을 수 있어요. 결제 알림을 켜두면 더 빨리 감지돼요."
