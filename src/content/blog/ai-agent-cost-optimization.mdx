# AI 에이전트 비용 관리: 같은 도구로 10배 절약하는 법

오픈클로 디스코드에 이런 글이 올라왔습니다.

> "하루에 $50 나가요. 도와주세요."

같은 디스코드에 다른 글도 있었습니다.

> "플러그인 하나 바꿨더니 월 $847에서 $127로 줄었어요."

같은 도구, 같은 기능인데 비용이 7배 차이납니다. 왜 그럴까요? 그리고 어떻게 하면 줄일 수 있을까요?

## 비용은 세 곳에서 만들어집니다

대부분은 "어떤 모델을 쓰느냐"가 비용을 결정한다고 생각합니다. 틀리지 않지만, 그게 전부가 아닙니다.

AI 에이전트 비용을 결정하는 요소는 세 가지입니다.

1. **모델 가격** — Opus는 Sonnet보다 6~10배 비쌉니다
2. **컨텍스트 길이** — 매번 얼마나 많은 내용을 모델에게 보내는가
3. **자동 실행 빈도** — 사용자가 보지 않는 곳에서 AI가 얼마나 자주 돌아가는가

대부분이 1번만 신경 씁니다. 진짜 돈은 2번과 3번에서 빠져나갑니다.

### 대화가 길어질수록 비용이 기하급수로 오릅니다

AI와 대화할 때, 매번 지금까지의 모든 대화 내용을 통째로 보냅니다. 첫 번째 메시지: 1페이지. 두 번째 메시지: 2페이지. 백 번째 메시지: 100페이지. 이게 컨텍스트 윈도우입니다.

GitHub Issue #9157을 보면, 오픈클로의 워크스페이스 파일(AGENTS.md, SOUL.md 등)만 매 메시지마다 **35,600토큰**씩 주입됩니다. 대화 100번이면 워크스페이스 파일에만 356만 토큰이 소비됩니다. 파일 내용은 전혀 안 바뀌는데도요.

### 하트비트가 조용히 돈을 씁니다

하트비트는 AI가 "나 살아있어요"를 주기적으로 확인하는 기능입니다. 문제는 매번 **전체 컨텍스트를 로드**한다는 겁니다.

GitHub Issue #43767의 보고:

> "하트비트가 lightContext 설정을 무시합니다. 매번 AGENTS.md, SOUL.md, TOOLS.md, IDENTITY.md, USER.md, HEARTBEAT.md, MEMORY.md 전부 주입합니다. 시스템 프롬프트만 7,000토큰. 5분마다 돌리면 하루에 200,000토큰 이상이 하트비트에만 소비됩니다."

보안 카메라가 있는데, 보안 카메라가 작동 중인지 확인하는 센서가 매번 CCTV 전체 녹화본을 다시 보고 있는 겁니다.

## 모델 라우팅: 85% 절감의 방법

모든 요리에 트러플 오일을 쓰면 파산합니다. AI도 마찬가지입니다.

앞서 언급한 $847 → $127 사례는 "자동 모델 라우팅 플러그인"을 설치한 결과입니다. 원리는 단순합니다. 작업 종류에 맞게 모델을 자동으로 선택합니다.

| 작업 유형 | 적합한 모델 | 상대 비용 |
|---------|------|-----------|
| 하트비트, 분류, 번역, 요약 | Flash-Lite / Haiku | 약 0.1배 |
| 일반 대화, 이메일 정리, 문서 작성 | Sonnet / GPT-5.3 | 1배 |
| 설계 판단, 복잡한 추론, 코드 리뷰 | Opus / GPT-5.4 | 6~10배 |

택시비가 아까워서 모든 이동에 리무진을 타다가, "지하철 + 가끔 택시" 전략으로 바꾸는 것과 같습니다. 결과는 똑같고, 비용은 80% 이상 줄어듭니다.

openclaw.json에서 설정하면 됩니다.

```json
{
  "agents": {
    "defaults": {
      "model": { "primary": "anthropic/claude-sonnet-4-5" },
      "heartbeat": { "model": "google/gemini-flash-lite" }
    }
  }
}
```

하트비트 모델만 분리해도 10~20% 절감이 가능합니다. 서브에이전트 기본을 Haiku로 설정하고 어려운 판단에만 Opus를 오버라이드하면 50~60% 절감까지 갑니다.

## 요금이 엉뚱한 곳에서 나가는 경우

GitHub Issue #39818에서 실제로 일어난 일입니다.

> "사용자가 GPT-5.3을 쓴다고 생각했는데, 실제로는 Kimi K2.5 API에 과금되고 있었습니다. API 청구서를 직접 확인하기 전까지는 알 수 없었습니다."

기본 모델이 에러나면 오픈클로는 조용히 다른 모델로 넘어갑니다. `/status`조차 폴백 후에도 원래 모델명을 표시하는 버그(#38567)가 있었습니다. 에어컨인 줄 알고 틀었는데 사실 전기히터였던 상황입니다.

모델 설정은 `/status`로 확인하고, API 제공자 대시보드에서 교차 검증하는 습관이 필요합니다.

## Thinking 토큰: 짧은 답변의 긴 청구서

AI가 짧은 답을 해도, 내부에서는 길게 "생각"할 수 있습니다. 그 생각하는 시간에도 과금됩니다.

변호사에게 "한 줄 답변"을 받았지만, 그 한 줄을 만들기 위한 3시간 리서치 비용이 청구되는 구조입니다. `thinking: "xhigh"`를 켜면 응답 품질은 올라가지만, 비용도 수 배로 뜁니다.

커뮤니티에서 보고된 사례:

- Google Ultra에서 하루 $1,600어치 토큰 소비
- Claude 프롬프트당 $1씩 나가는 경우

Thinking 레벨은 실제로 필요한 작업에만 적용하는 것이 합리적입니다.

## 컴팩션도 공짜가 아닙니다

대화가 길어지면 비용이 폭발합니다. 이걸 막는 게 컴팩션(대화 요약 압축)입니다. 그런데 컴팩션 자체에도 비용이 있습니다.

VelvetShark Memory Masterclass의 핵심 지적:

> "컴팩션이 발생하면 프롬프트 캐시가 무효화됩니다. 다음 요청에서 전체 재캐싱 비용이 발생합니다."

| 전략 | 비용 특성 | 기억 |
|-----|------|------|
| 컴팩션 안 함 | 기하급수로 상승 | 전부 기억 |
| 자동 컴팩션 | 낮게 유지되지만 캐시 미스 발생 | 세부 내용 손실 |
| 수동 컴팩션 + 메모리 플러시 | 최적화 가능 | 핵심만 기억 |

컴팩션 빈도와 기억 보존은 트레이드오프입니다. 어느 쪽을 우선시할지 전략적으로 결정해야 합니다. 컴팩션에 대한 자세한 내용은 [AI의 기억상실: 컴팩션은 왜 필요하고 뭘 잃는가](/blog/ai-memory-compaction)에서 다룹니다.

## Claude Max 구독인데 왜 돈이 나가죠?

커뮤니티에서 가장 자주 나오는 혼란입니다.

> "Claude Max 구독($200/월) 결제했는데 왜 API 요금이 또 나오죠?"

Claude Max는 Claude.ai 웹/앱 사용량만 커버합니다. 오픈클로는 Anthropic API를 직접 호출하므로 별도 과금됩니다.

2026년 2월부터 OAuth 토큰으로 Max 구독을 오픈클로에 연결할 수 있게 되었습니다. 하지만 rate limit이 있어서 자주 쓰는 환경에서는 API 키가 더 유리합니다.

넷플릭스 프리미엄을 결제했는데 특정 기기에서 추가 요금이 나간다고 생각했는데, 사실 설정을 바꾸면 프리미엄 안에서 쓸 수 있었던 것과 비슷합니다. 단, 이용량에 한도가 있습니다.

## 비용 절감 체크리스트

당장 확인해볼 항목들입니다.

1. **하트비트 모델 분리** — 하트비트에 Sonnet이나 Opus가 할당되어 있다면 즉시 교체
2. **모델 라우팅 설정** — 작업 유형별로 적합한 모델 자동 선택
3. **thinking 레벨 검토** — `xhigh`가 정말 필요한 작업인지 판단
4. **폴백 모델 확인** — `/status`와 API 대시보드를 함께 보기
5. **워크스페이스 파일 크기** — AGENTS.md가 비대하면 매 메시지마다 비용 증가
6. **컴팩션 전략 설정** — 수동 컴팩션 타이밍 정하기
7. **구독 활용** — Claude Max OAuth 연결 가능 여부 확인
