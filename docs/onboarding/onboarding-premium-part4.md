### Step 1) GitHub에 코드 올리기 (7분)

> "git은 코드의 타임머신이에요. 언제든 이전 버전으로 돌아갈 수 있어요."

아래 명령어를 **순서대로** 실행해요.

```bash
git init
git add .
git commit -m "first commit"
# GitHub에서 새 repo 만들기 → 주소 복사
git remote add origin <URL>
git push -u origin main
```

#### 명령어 한 줄 설명 (비개발자 버전)

- `git init` → 이 폴더를 Git으로 버전관리 시작해요.
- `git add .` → 지금 변경된 파일을 "저장 후보"로 올려요.
- `git commit -m "first commit"` → 변경 내역에 이름 붙여 한 번 저장해요.
- `git remote add origin <URL>` → 내 컴퓨터 Git과 GitHub 저장소를 연결해요.
- `git push -u origin main` → 내 코드를 GitHub에 업로드해요.

#### 브랜치 이름 이슈 빠른 해결 (필요 시)

일부 환경에서 기본 브랜치가 `master`일 수 있어요. 에러가 나면 아래 먼저 실행해요.

```bash
git branch -M main
git push -u origin main
```

#### Cursor에서 Git 기능도 같이 보여주기 (Source Control 탭)

> "터미널이 낯설면 Cursor에서도 거의 같은 걸 클릭으로 할 수 있어요."

1. 좌측 사이드바에서 **Source Control(브랜치 아이콘)** 클릭해요.
2. 변경된 파일 목록 확인해요.
3. 상단 메시지 입력칸에 커밋 메시지 입력해요. (예: `first commit`)
4. **Commit** 버튼 클릭해요.
5. **Publish Branch** 또는 **Push** 클릭해 GitHub로 업로드해요.

> "터미널 방식, 클릭 방식 둘 다 가능해요. 편한 걸 쓰시면 돼요."

---

### Step 2) Vercel 연결해서 배포하기 (4분)

> "이제 GitHub에 올린 코드를 Vercel에 연결하면 자동으로 배포돼요."

1. `https://vercel.com` 접속해 로그인해요.
2. 대시보드에서 **New Project** 클릭해요.
3. 방금 만든 GitHub repo를 선택해요.
4. Framework Preset이 Next.js로 잡혀 있는지 확인해요.
5. **Deploy** 클릭해요.
6. 1~2분 기다리면 URL이 생성돼요. (예: `https://my-todo.vercel.app`)

> "축하합니다! 방금 만든 앱이 인터넷에 올라갔어요."

✋ [고객 실습]

- 본인 폰으로 배포된 URL에 직접 접속해요.
- Todo 추가/삭제를 한 번 눌러봐요.
- 함께 말해줘요: **"본인이 만든 앱이에요!"**

> "진짜 중요해요. 지금 본인이 만든 앱이 전 세계 어디서든 열려요."

---

### Step 3) 커스텀 도메인 연결 (선택, 3분)

> "도메인이 있으시면 연결할 수 있어요. 없으시면 .vercel.app 주소로도 충분해요."

1. Vercel 프로젝트 → **Settings** → **Domains**로 이동해요.
2. 보유 도메인 입력 후 **Add** 클릭해요.
3. 안내되는 DNS 레코드를 도메인 구매처(가비아/Namecheap 등)에 추가해요.
4. 연결 완료까지 보통 몇 분~수십 분 걸릴 수 있어요.

> "처음엔 무료 기본 도메인으로 시작하고, 나중에 브랜드명 도메인 붙이면 돼요."

## 고객 예상 질문 + 답변 (FAQ)

### Q1. "배포한 앱에 다른 사람이 들어오면 내 할 일이 보이나요?"
A. "아니요. 지금은 `localStorage`를 써서 각자 브라우저에만 저장돼요. 같은 URL이어도 사람마다 데이터가 달라요."

### Q2. "Vercel 무료인가요?"
A. "네, Hobby 플랜은 무료예요. 개인 프로젝트에는 충분해요."

### Q3. "도메인은 어디서 사나요?"
A. "가비아(gabia.com), Namecheap 같은 곳에서 보통 연 1~2만 원대로 구매할 수 있어요."

### Q4. "앱이 갑자기 안 되면요?"
A. "Vercel 대시보드의 Deployments 먼저 확인해요. 최근 배포가 실패했다면 이전 정상 배포로 되돌릴 수 있어요."

### Q5. "데이터베이스는 안 쓰나요?"
A. "오늘은 브라우저 저장(localStorage)만 썼어요. 나중에 Supabase/Firebase를 붙이면 계정 기반 앱으로 확장할 수 있어요."

### Q6. "수정할 때마다 Vercel에서 뭘 다시 눌러야 하나요?"
A. "아니요. `git push`만 하면 자동 배포돼요. Vercel은 연결 후 대부분 자동이에요."

### Q7. "실수로 코드 망가뜨리면 끝인가요?"
A. "괜찮아요. Git 커밋 기록이 있어서 이전 버전으로 복구할 수 있어요. 그래서 커밋을 자주 하는 게 중요해요."

### Q8. "팀원/친구와 같이 개발할 수도 있나요?"
A. "가능해요. GitHub에 같이 협업하면 돼요. 나중에 branch와 pull request 방식까지 배우면 실무 방식으로 작업할 수 있어요."

### Q9. "배포했는데 폰에서 레이아웃이 깨져요."
A. "모바일 반응형이 덜 된 상태예요. Cursor에 `모바일 390px 기준으로 레이아웃 수정`처럼 구체적으로 요청하고 다시 push하면 바로 반영돼요."

### Q10. "내 앱 주소를 포트폴리오에 써도 되나요?"
A. "당연해요. 오늘 만든 URL 자체가 이미 결과물이자 포트폴리오예요. 이게 첫 번째 공개 프로젝트예요."
