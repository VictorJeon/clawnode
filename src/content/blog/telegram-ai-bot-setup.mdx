# 텔레그램 AI 봇 만들기: 오픈클로로 24시간 비서 세팅

스마트폰에서 텔레그램을 열고, 메시지 하나 보냅니다. "오늘 일정 정리해줘." 그러면 AI가 캘린더를 확인하고, 오늘 미팅 목록을 정리해서 돌려줍니다. 파일 열지 않아도, 컴퓨터 앞에 앉지 않아도.

이게 오픈클로 + 텔레그램 조합으로 만들 수 있는 환경입니다. 오픈클로가 집 컴퓨터(맥미니)에서 24시간 실행되고 있고, 텔레그램이 그 컴퓨터로 접속하는 창구가 됩니다.

세팅은 크게 세 단계입니다. BotFather로 봇 생성 → 오픈클로에 토큰 입력 → 대화 시작.

## 1단계: BotFather로 봇 만들기

텔레그램에서 모든 봇은 BotFather를 통해 만들어집니다. 텔레그램 앱에서 `@BotFather`를 검색하고 채팅을 시작합니다.

BotFather에게 `/newbot`을 보내면 이름과 username을 물어봅니다.

```
/newbot
```

이름은 봇의 표시 이름입니다. `My AI Assistant` 같은 자유로운 이름을 쓰면 됩니다.

username은 고유 식별자입니다. 반드시 `bot`으로 끝나야 합니다. `myaiassistant_bot`처럼요.

설정이 완료되면 BotFather가 **API 토큰**을 발급해줍니다. 이렇게 생긴 긴 문자열입니다.

```
1234567890:ABCDEFghijklmnopqrstuvwxyz-12345678
```

이 토큰이 봇의 열쇠입니다. 외부에 노출하면 안 됩니다.

## 2단계: 오픈클로 설정 파일에 토큰 입력

오픈클로는 텔레그램 채널 플러그인을 지원합니다. `openclaw.json` 설정 파일에 토큰을 추가하면 됩니다.

오픈클로가 설치된 디렉토리에서 `openclaw.json`을 열고, `channels` 항목에 텔레그램 설정을 추가합니다.

```json
{
  "channels": {
    "telegram": {
      "token": "여기에-BotFather-토큰-붙여넣기",
      "allowedUsers": [123456789]
    }
  }
}
```

`allowedUsers`는 본인 텔레그램 계정 ID입니다. 텔레그램에서 `@userinfobot`에게 메시지를 보내면 본인 ID를 알 수 있습니다. 이걸 설정하지 않으면 누구나 이 봇에 접근할 수 있으니 반드시 채워야 합니다.

수정 후 오픈클로를 재시작합니다.

```bash
openclaw gateway restart
```

## 3단계: 텔레그램에서 봇 찾아서 대화 시작

BotFather가 알려준 봇 username으로 검색합니다. `@myaiassistant_bot`처럼요.

채팅창을 열고 `/start`를 보내면 연결됩니다.

이제 그냥 말을 걸면 됩니다.

```
이번 주 금요일 오후 일정 비워놔
```

```
어제 받은 이메일 중에 회신 필요한 것 있어?
```

오픈클로가 설정된 도구들(구글 캘린더, 이메일, 파일 시스템 등)을 써서 답합니다. 스마트폰에서 보내도, 텔레그램 데스크탑에서 보내도 동일하게 작동합니다.

## 이 구조가 왜 편리한가

맥미니는 집에서 항상 켜져 있습니다. 오픈클로는 그 위에서 계속 실행됩니다. 텔레그램은 어디서든 접근할 수 있습니다.

회의 중에 스마트폰으로 "다음 주 월요일 미팅 자료 준비해줘"를 보내면, 집 컴퓨터에 있는 오픈클로가 파일을 정리하고 구글 드라이브에 올려줍니다. 본인이 집에 없어도 됩니다.

Tailscale이 설정돼 있으면 VPN 없이도 외부에서 안전하게 접속됩니다. 이 설정은 [오픈클로 설치 가이드](/blog/openclaw-install-guide)에 상세히 나와 있습니다.

## 자주 묻는 것들

**봇이 응답을 안 해요.**
`openclaw gateway status`로 게이트웨이가 실행 중인지 확인하세요. 토큰이 제대로 복사됐는지도 확인해야 합니다. 공백이 들어가면 인증 오류가 납니다.

**가족이 봇에 접근할 수 있나요?**
`allowedUsers`에 본인 ID만 넣으면 다른 사람은 메시지를 보내도 응답하지 않습니다.

**프라이빗 그룹에 추가할 수 있나요?**
가능합니다. 봇을 그룹에 추가하고 관리자 권한을 주면 됩니다. 다만 그룹 내 모든 메시지를 수신하므로, `allowedUsers` 설정이 더 중요해집니다.
