# 오픈클로 vs ChatGPT: 뭐가 다르고 언제 쓸까

둘 다 AI이고, 둘 다 한국어로 대화할 수 있고, 둘 다 "일을 도와주는" 도구입니다.

그런데 실제로 쓰는 방식은 완전히 다릅니다.

ChatGPT는 채팅창을 열고 질문합니다. 오픈클로는 내 컴퓨터에서 자율적으로 실행됩니다. 이 차이가 무엇을 의미하는지, 어떤 상황에서 어떤 걸 쓸지 정리해봤습니다.

## 작동 방식의 차이

ChatGPT는 OpenAI 서버에서 실행되는 채팅 서비스입니다. 브라우저나 앱에서 질문하면, 그 내용이 OpenAI 서버로 전송되고, 모델이 답변을 생성해서 돌려줍니다. 탭을 닫으면 연결이 끊깁니다. 다음번에 새 대화를 시작하면 이전 맥락은 없습니다.

오픈클로는 내 컴퓨터(예: 맥미니)에 설치되어 24시간 실행되는 AI 에이전트입니다. 파일 시스템, 이메일, 캘린더, 외부 API에 직접 접근할 수 있습니다. 사용자가 자리를 비워도 예약된 작업을 처리하고, 텔레그램으로 원격 지시도 받습니다.

## 핵심 비교표

| 항목 | ChatGPT | 오픈클로 |
|------|---------|---------|
| 실행 위치 | OpenAI 서버 (클라우드) | 내 컴퓨터 (로컬) |
| 데이터 처리 | 외부 서버로 전송 | 로컬에서 처리 |
| 작동 시간 | 브라우저/앱 사용 중만 | 24시간 상시 실행 |
| 내 파일 접근 | 수동 업로드 필요 | 직접 접근 가능 |
| 예약 작업 | 불가 | 크론으로 스케줄 가능 |
| 원격 제어 | 앱으로 접속 필요 | 텔레그램으로 어디서나 |
| 커스터마이즈 | 제한적 | SOUL.md로 완전 커스터마이즈 |
| 월 비용 | $20~$200+ | API 비용만 (수천~수만원) |
| 장기 기억 | 제한적 (메모리 기능) | MEMORY.md 기반 장기 기억 |

## 용도가 다릅니다

**ChatGPT가 맞는 상황:**

- 즉석에서 질문하고 답변받기
- 글 초안 잡기, 아이디어 브레인스토밍
- 코드 한 두 줄 물어보기
- 빠르게 번역이나 요약이 필요할 때
- 팀원 모두가 쓸 수 있는 간단한 도구로

**오픈클로가 맞는 상황:**

- 이메일, 파일, 캘린더를 직접 다루는 업무 자동화
- 매일 반복되는 작업을 자동으로 처리하고 싶을 때
- 민감한 데이터를 외부 서버에 보내고 싶지 않을 때
- 스마트폰에서 원격으로 컴퓨터 작업을 시키고 싶을 때
- 나만의 업무 스타일을 기억하는 AI를 원할 때

## 비용은 어떻게 다른가

ChatGPT Plus는 월 $20입니다. ChatGPT Team은 인당 $25~$30입니다. 더 강력한 기능이 필요하면 ChatGPT Pro가 월 $200입니다.

오픈클로 자체는 무료 오픈소스입니다. LLM API 비용만 냅니다. Claude Sonnet을 연동해서 하루 100회 요청한다고 가정하면, 한 달 API 비용은 약 3~5달러 수준입니다. 맥미니 초기 구매비용(약 90만원)이 있지만, ChatGPT Pro 기준으로 4~5개월이면 회수됩니다.

하지만 비용만으로 비교하는 건 의미가 없습니다. 오픈클로가 처리하는 업무의 복잡도가 다르기 때문입니다.

## 보안에서의 차이

ChatGPT에 회사 내부 문서, 고객 데이터, 계약서 내용을 붙여넣는 건 위험합니다. OpenAI 서버를 거치기 때문입니다. 대부분의 대기업 보안 정책에서 이를 금지하는 이유입니다.

오픈클로는 로컬에서 실행됩니다. Ollama 같은 로컬 LLM을 연동하면 추론 단계까지 외부 서버를 거치지 않습니다. 민감한 데이터를 다뤄야 하는 법무, 회계, 인사 업무에서 유리합니다. [로컬 AI의 장점](/blog/local-ai-privacy)에서 이 주제를 더 자세히 다룹니다.

## 둘을 같이 쓰는 방법도 있습니다

오픈클로와 ChatGPT는 경쟁 관계가 아닙니다.

오픈클로 안에서 OpenAI GPT 모델을 API로 연동해서 쓸 수 있습니다. 즉, 오픈클로가 자율적으로 작업을 처리하면서 필요한 순간에 GPT-4o나 Claude를 호출하는 구조입니다.

가장 좋은 접근은 "뭘 대화로 처리하고, 뭘 자동화할지"를 나누는 겁니다.

- 창의적 작업, 즉흥적 질문 → ChatGPT 
- 반복 업무, 로컬 파일/도구 연동 → 오픈클로
