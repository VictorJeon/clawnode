### Part 1 예상 질문 & 답변 (PREMIUM)

**Q1. 프롬프트를 잘 못 써도 괜찮나요?**
- 네, 괜찮아요. 내비게이션에 목적지만 말해도 길을 안내하듯, AI도 핵심 목적만 명확하면 됩니다.

**Q2. 답이 이상하면 제가 뭘 잘못한 건가요?**
- 대부분은 요청 범위가 넓어서 그래요. 카메라 초점 맞추듯 범위를 좁히면 품질이 바로 올라가요.

**Q3. 파일 분석은 어디까지 믿어도 되나요?**
- 초안/정리는 매우 강해요. 다만 계약/법률/숫자 확정은 최종 검토가 필요해요. AI는 빠른 보조 엔진으로 쓰는 게 맞아요.

**Q4. 음성으로만 써도 실무에 충분한가요?**
- 충분해요. 다만 음성 하나에 주제 하나가 좋아요. 택배 상자에 물건을 섞지 않는 것과 같아요.

**Q5. 회사 정보 넣어도 되나요?**
- 민감정보는 마스킹하고 핵심만 넣는 걸 권장해요. 주민번호를 별표 처리하듯 최소한으로 공유하면 안전해요.

**Q6. 이걸 매일 쓰려면 루틴이 필요할까요?**
- 네. 아침 1회 요약, 점심 전 1회 초안, 퇴근 전 1회 정리 루틴이면 체감이 큽니다.

## [30:00~40:00] Ch4. 바이브코딩이란? (10분)

### [31:00~34:00] 개념 설명
🖥️ [화면 공유] 슬라이드: "자연어 → 코드 → 실행"

> 바이브코딩은 말로 지시해서 앱을 만드는 방식이에요.

> 요리 비유로 보면, 레시피를 직접 코딩하는 대신 셰프에게 "짭짤한 파스타 만들어줘"라고 요청하는 거예요.

> 핵심은 세 가지예요.
>
> 첫째, 프로그래밍을 몰라도 됩니다.
>
> 둘째, 중요한 건 "무엇을 만들지"를 명확히 설명하는 능력이에요.
>
> 셋째, AI가 80%를 만들고, 나머지 20%는 "이 버튼 더 크게", "색상 바꿔줘"처럼 수정 지시로 완성합니다.

## Part 2 예상 질문 & 답변

**Q1. 진짜 코드를 몰라도 앱을 만들 수 있나요?**
- 네, 시작은 충분히 가능해요. 발표 자료를 직접 디자인툴 없이 템플릿으로 만드는 것과 비슷해요. 다만 고급 기능은 점진적으로 배우면 더 좋아요.

**Q2. AI가 만든 코드가 틀릴까 봐 걱정돼요.**
- 맞아요, 100% 정답은 아니에요. 그래서 "생성 → 실행 → 수정 지시" 3단계를 반복해요. 초안 작성 후 교정하는 문서 작업과 같아요.

**Q3. 어떤 앱이 바이브코딩에 가장 잘 맞나요?**
- 예약폼, 관리자 페이지, 사내 도구, 랜딩페이지가 가장 잘 맞아요. 복잡한 결제/보안/대규모 트래픽 서비스는 전문가 협업이 필요해요.

**Q4. Cursor 유료 결제는 꼭 해야 하나요?**
- 처음에는 체험으로 충분해요. 사용 빈도가 주 3회 이상이면 유료가 효율적이에요. 헬스장도 자주 가는 사람이 회원권이 이득인 것과 같아요.

**Q5. GitHub는 왜 꼭 필요해요?**
- 백업 + 버전관리 + 배포 연결을 한 번에 해결해줘요. USB 하나에만 저장하면 불안한데, 클라우드에 기록이 남으면 안전한 것과 같아요.

**Q6. Node 버전이 안 나오면 어떻게 하나요?**
